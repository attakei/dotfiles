==================
管理レイヤーの分類
==================

リポジトリが管理しているものは、
実際の利用を踏まえると複数の **「レイヤー」** で切り離して考えるのが望ましい。
このページでは、各レイヤーの役割や責務について記載する。

全体像
======

おおざっぱには、下記のような4階層で整理する。

+-------------------+
| Application layer |
+-------------------+
| Shell layer       |
+-------------------+
| Package layer     |
+-------------------+
| OS layer          |
+-------------------+

OS layer
========

Package layerのために使用するツールを安定的に使用可能とするために、
あらかじめ実施しないといけない処理をするための層。

対象の例:

* aquaのグローバル設定
* aqua, Scoopのインストール
* Shell layer向けの環境変数

手段の例:

* Powershell
* Bash
* curl

Package layer
=============

dotfiles配下やdotfilesを前提とした運用体制で実行するソフトウェアを管理するための層。

対象の例:

* aquaの利用
* Scoopの利用(Windowsのみ)

手段の例:

* Powershell(初回)
* Bash(初回)
* Zsh
* Nushell

Shell layer
===========

通常利用するシェルの動作に影響を与えるものを管理するための層。

対象の例:

* Nushellの設定
* Zshの設定
* Zellij等のツール群向けの環境変数

手段の例:

* テキスト
